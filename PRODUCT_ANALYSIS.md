## 1️⃣ 앱의 정체성과 핵심 목표

### 🎯 정체성

**"점진적 과부하 추적을 통한 데이터 기반 운동 성장 플랫폼"**

#### 타겟 사용자
- 웨이트 트레이닝을 하는 운동 초보자 ~ 중급자
- 자신의 운동 기록을 체계적으로 관리하고 싶은 사람
- 진척도를 시각적으로 확인하며 동기부여를 받고 싶은 사람
- 운동 루틴을 공유하고 다른 사람의 루틴을 참고하고 싶은 사람

#### 핵심 가치 (Core Values)
1. **점진적 과부하 추적 (Progressive Overload)**
2. **실시간 피드백 (Real-time Feedback)**
3. **스냅샷 기반 영속성 (Snapshot-based Persistence)**

---

### 🏆 가장 큰 목표

#### 1. 점진적 과부하 추적 (Progressive Overload)
**목표:**
- 동일 루틴 반복 시 회차별 성장 비교
- "지난번보다 무거운 무게를 들었나?" → 핵심 동기부여 요소

**현재 구현:**
- `GET /api/workouts/sessions/routines/{routineId}/last` - 직전 회차 조회
- 홈 화면 통계 (이번 주/달, 주간 활동, 자주 하는 운동 TOP 5)

**미완성:**
- 전체 회차 비교 통계 (Phase 2: 20% 완료)

---

#### 2. 실시간 피드백
**목표:**
- 루틴 수행 시 직전 기록 참고하여 목표 설정
- "저번엔 60kg 10회 → 이번엔 더 할 수 있나?" 심리적 자극

**현재 구현:**
- 운동 세션 시 이전 운동 데이터 자동 로드
- 세트별 무게/횟수 입력 시 실시간 볼륨 계산

---

#### 3. 데이터 무결성
**목표:**
- 스냅샷 기반 영속성: 루틴/종목 수정해도 과거 기록 불변
- 운동 이력의 신뢰성 보장

**현재 구현:**
```sql
workout_sets:
  - exercise_name (스냅샷)
  - body_part_snapshot (스냅샷)
  - exercise_id (참조용, CASCADE 없음)
```

### D. 비즈니스 로직 측면 ✅

#### 세션 완료 액션 정책 (독창적)

```
사용자 선택 → 자동 처리:

1. RECORD_ONLY
   → 기록만 저장

2. CREATE_ROUTINE_AND_RECORD (자유 운동 전용)
   → 세트 기록 기반 새 루틴 자동 생성

3. DETACH_AND_RECORD (루틴 기반 전용)
   → 자유 운동으로 전환

4. UPDATE_ROUTINE_AND_RECORD (루틴 기반 전용)
   → 기존 루틴 구조 자동 업데이트
```

**차별화:** 대부분의 앱은 "저장" 하나만 제공. PROLOG는 사용자 의도에 맞는 4가지 옵션 제공.

## 3️⃣ 부족한 부분 및 개선 제안

### A. 기획/제품 측면 ⚠️

#### 1. 홈 화면 통계의 깊이 부족

**현황:**
- Phase 2에서 통계 API 5개 중 1개만 완료 (20%)
- 종목별 볼륨 추이, 최고 중량 추이, 루틴별 회차 비교 미구현

**문제점:**
- 핵심 가치인 "점진적 과부하 추적"이 통계 부족으로 약화
- 홈 화면이 단순 기록 조회 수준

**개선 제안:**

```markdown
📊 추가 필요 통계:

**Phase 2 완성 (필수):**
1. 종목별 볼륨 추이 (일/주/월)
2. 종목별 최고 중량 추이
3. 루틴별 회차 비교 (1회 vs 2회 vs 3회...)
4. 운동 요약 대시보드

**Phase 2+ (고급):**
1. 종목별 1RM 추정 (Brzycki 공식)
   - 1RM = weight × (1 + reps/30)
2. 부위별 밸런스 차트 (레이더 차트)
   - 가슴 vs 등 vs 하체 vs 어깨 등
3. 월간/연간 총 볼륨 트렌드
4. 운동 빈도 히트맵 (캘린더 뷰)
5. PR (Personal Record) 뱃지 시스템
   - "벤치프레스 80kg 달성!" 알림
```

**우선순위:** ⭐⭐⭐⭐⭐ (최우선)

**파일 위치:**
- Backend: `backend/src/main/java/com/back/domain/stats/`

#### 3. 운동 중 UX 개선점

**현황:**
- 세트 입력, 완료 체크, 타이머만 제공
- 휴식 시간 관리 부재

**문제점:**
- 휴식 시간 타이머 없음 → 사용자가 시계 앱 별도 사용
- 세트 완료 시 피드백 부족 → 완료 실감 부족
- 운동 순서 변경이 번거로움 (롱프레스 → 위/아래 버튼)

**개선 제안:**

```markdown
🏋️ 운동 세션 UX 개선:

1. 자동 휴식 타이머
   ├─ 세트 완료 체크 시 자동 시작
   ├─ 설정한 휴식 시간 카운트다운
   ├─ 타이머 종료 시 진동 + 소리 알림
   └─ "건너뛰기" 버튼 (즉시 다음 세트)

2. 세트 완료 피드백
   ├─ 체크 시 햅틱 피드백 (진동)
   ├─ 애니메이션 (체크마크 확대)
   └─ 완료 세트 수 실시간 표시

3. Drag & Drop 운동 순서 변경
   ├─ react-native-draggable-flatlist
   └─ 롱프레스 후 드래그로 순서 변경

4. 운동 건너뛰기
   ├─ 특정 운동 일시 제외 (부상 등)
   └─ 기록에는 남지 않음

5. 볼륨 계산 실시간 표시
   ├─ 세트 입력 중 총 볼륨 자동 계산
   ├─ "현재 총 볼륨: 2,400kg" 표시
   └─ 직전 세션과 비교 (+200kg ↑)

6. 세트 노트
   ├─ 각 세트에 메모 추가
   └─ "이 세트 힘들었음", "폼 좋았음" 등
```

**우선순위:** ⭐⭐⭐⭐⭐ (최우선)

- 네트워크 에러 시 재시도 방법 불명확

**개선 제안:**

```markdown
🔔 피드백 시스템:

1. Toast 라이브러리 도입
   → react-native-toast-message

2. 성공 토스트
   ├─ "루틴이 생성되었습니다" (초록)
   ├─ "운동이 완료되었습니다" (초록)
   └─ "루틴을 가져왔습니다" (초록)

3. 에러 토스트
   ├─ "네트워크 오류. 다시 시도해주세요" (빨강)
   ├─ "이미 가져온 루틴입니다" (주황)
   └─ "권한이 없습니다" (빨강)

4. 로딩 스켈레톤 UI
   ├─ ActivityIndicator 대신
   └─ react-native-skeleton-placeholder

5. Retry 버튼
   └─ 네트워크 에러 시 화면에 "재시도" 버튼 표시
```

**우선순위:** ⭐⭐⭐⭐

**예시 코드:**
```tsx
import Toast from 'react-native-toast-message';

// 성공

  type: 'success',
  text1: '루틴이 생성되었습니다',
});

// 에러

  type: 'error',
  text1: '네트워크 오류',
```

2. **유연한 운동 플로우**
   - 루틴 기반 + 자유 운동 병행
   - 4가지 완료 액션 정책

3. **체계적인 문서화**
   - REQUIREMENTS.md (비즈니스 로직)
   - API.md (39개 엔드포인트)
   - DEVELOPMENT.md (개발 가이드)

4. **견고한 아키텍처**
   - Monorepo 구조
   - JWT + BCrypt 보안
   - Redis 캐싱 전략
